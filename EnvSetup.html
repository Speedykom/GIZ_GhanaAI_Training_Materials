Part 1: uv (by Astral)

What is uv?

uv is an extremely fast Python package and project manager created by Astral. It’s currently the hottest tool in the Python ecosystem, and for good reason.

NoteCompany vs. Product

The company is called Astral, and the tool is called uv. People sometimes confuse the two, so remember: Astral makes uv.

Think of uv as your Python “super-manager.” It’s written in Rust, which makes it incredibly fast—10 to 100 times faster than older Python tools.

What Does uv Do?

uv handles three critical tasks:

  1. Installs Python Itself
    You no longer need to download Python installers from python.org. uv can install and manage multiple Python versions for you.

  2. Manages Dependencies
    When building AI applications, we use libraries (code written by others) like TensorFlow, PyTorch, or scikit-learn. uv downloads and manages these for us.

  3. Creates Virtual Environments
    Virtual environments are isolated “sandboxes” for your projects. This means Project A can use Python 3.11 while Project B uses Python 3.12, and they won’t interfere with each other.

Understanding Virtual Environments: Why Do We Need Them?

Imagine you’re working on two Python projects:

Project A (Old Project): Built last year, uses Django 3.2 and Python 3.9
Project B (New Project): Started today, uses Django 5.0 and Python 3.12

The Problem Without Virtual Environments:

If you install packages globally on your computer, you can only have ONE version of each package installed at a time. This creates chaos:

  • Installing Django 5.0 for Project B breaks Project A
  • Downgrading to Django 3.2 for Project A breaks Project B
  • Different projects fight over which Python version to use
  • Your system Python (used by your operating system) can get corrupted
WarningReal-World Horror Story

A student once installed a package globally that conflicted with their system Python. Their computer’s software update system stopped working. They had to reinstall their entire operating system. Don’t let this be you.

The Solution: Virtual Environments

A virtual environment is like a separate “bubble” or “container” for each project:

  • Project A gets its own bubble with Django 3.2 and Python 3.9
  • Project B gets its own bubble with Django 5.0 and Python 3.12
  • Your system Python stays completely untouched and safe
  • Switch between projects instantly without breaking anything

Think of it like having separate toolboxes: your woodworking toolbox doesn’t mix with your electronics toolbox. Each project gets exactly what it needs, nothing more, nothing less.

Why This Matters for AI Development:

AI and machine learning libraries are notorious for version conflicts: - TensorFlow 2.10 might require NumPy 1.23 - PyTorch 2.0 might require NumPy 1.24 - Without virtual environments, you can’t work on both projects

Virtual environments let you experiment freely without fear of breaking anything.

pyproject.toml vs. requirements.txt: The Modern Standard

Let’s understand why pyproject.toml is superior and why uv uses it by default.

The Old Way: requirements.txt

For decades, Python projects used requirements.txt to list dependencies:

numpy==1.24.3
pandas==2.0.1
scikit-learn==1.2.2

Problems with requirements.txt:

  1. No Metadata: Where’s your project name? Version? Author? Description? You need separate files for that.

  2. No Development vs. Production Split: You have to manually create requirements-dev.txt and requirements-prod.txt.

  3. No Dependency Groups: Can’t easily say “these are for testing, these are for docs, these are for production.”

  4. No Tool Configuration: Where do you configure pytest? black? mypy? In separate files: .flake8, pytest.ini, setup.cfg… it’s a mess.

  5. Not a Standard: Just a convention. Every tool interprets it differently.

  6. No Build System Info: If you want to publish your package to PyPI, you need additional files (setup.py, setup.cfg, MANIFEST.in).

  7. Version Locking Issues: requirements.txt typically uses exact versions (torch==2.1.0), which means any security update or bug fix requires manual changes. If PyTorch 2.1.1 fixes a critical bug, you’re stuck until you manually update the file.

The New Way: pyproject.toml

pyproject.toml is the official Python standard (defined in PEP 518, 621, 660) for project configuration. It’s a single file that contains everything:

[project]
name = "my-ai-project"
version = "0.1.0"
description = "My first AI application with PyTorch"
authors = [
    {name = "speedykom", email = "speedykom@example.com"}
]
requires-python = ">=3.11"
dependencies = [
    "torch>=2.1.0",
    "torchvision>=0.16.0",
    "numpy>=1.24.0",
]

[project.optional-dependencies]
dev = [
    "pytest>=7.0.0",
    "black>=23.0.0",
]

[tool.pytest.ini_options]
testpaths = ["tests"]

[tool.black]
line-length = 88

[build-system]
requires = ["hatchling"]
build-backend = "hatchling.build"

Why pyproject.toml is Better:

  1. Single Source of Truth
    Everything about your project in one file. No hunting through 5 different config files.

  2. Rich Metadata
    Project name, version, authors, license, Python version requirements—all in one place.

  3. Dependency Groups
    Separate development dependencies from production dependencies clearly:

    dependencies = ["numpy"]  # Production
[project.optional-dependencies]
dev = ["pytest"]  # Development only

  4. Tool Configuration
    Configure all your tools in one file:

    [tool.pytest.ini_options]
[tool.black]
[tool.mypy]

  5. Official Standard
    Defined by Python Enhancement Proposals (PEPs). Every modern Python tool supports it.

  6. Version Ranges
    Specify flexible version requirements:

    dependencies = [
    "torch>=2.1.0,<3.0.0",      # Any 2.x version from 2.1+
    "torchvision>=0.16.0",       # Any version 0.16 or newer
]

    This means if PyTorch 2.1.1 comes out with bug fixes, your project automatically gets them without manual updates.

  7. Works with Lock Files
    uv creates a uv.lock file that pins exact versions for reproducibility, while pyproject.toml specifies version ranges for flexibility.

Real-World Example

Scenario: You’re building an AI computer vision app. You need PyTorch for production, but pytest for testing.

With requirements.txt:

requirements.txt:
torch==2.1.0
torchvision==0.16.0

requirements-dev.txt:
torch==2.1.0
torchvision==0.16.0
pytest==7.4.0

You have to maintain two files and keep PyTorch versions in sync manually.

With pyproject.toml:

[project]
dependencies = ["torch>=2.1.0", "torchvision>=0.16.0"]

[project.optional-dependencies]
dev = ["pytest>=7.4.0"]

One file, clear separation, flexible versioning. When PyTorch 2.1.1 is released, you get it automatically.

NoteThe Industry is Moving

Major Python projects (Django, FastAPI, Pandas, NumPy) have all migrated to pyproject.toml. Learning it now means you’re learning the standard that will be used for decades to come.

How uv Uses pyproject.toml

When you create a project with uv:

uv init my-project

It automatically creates a pyproject.toml with sensible defaults.

When you want to add any other library to your project in the future a simple uv add <library name> adds that library to the dependencies. Also, when you want to share the same project with someone, they’d need the folder with the pyproject.toml file and all they’d need to do is cd into the folder and then run the code uv sync which installs everything listed in pyproject.toml.

It’s elegant, modern, and fast.

🖥️ Windows Installation (Recommended Method: Winget)

Winget is Windows’ built-in package manager. It’s available on Windows 10 (version 1809+) and Windows 11 by default.

Steps:

  1. Open PowerShell as Administrator

    • Press the Windows Key on your keyboard
    • Type “PowerShell”
    • Right-click “Windows PowerShell” and select “Run as administrator”
    • Click “Yes” if prompted by User Account Control

  2. Install uv using Winget

    Copy and paste this command into PowerShell and press Enter:

    winget install --id=astral-sh.uv -e

  3. Restart Your Terminal

    Close PowerShell completely and open a new PowerShell window. This ensures your system recognizes the uv command.

  4. Verify Installation

    Type this command and press Enter:

    uv --version

    You should see something like: uv 0.4.30 (version number may vary)

WarningIf Winget Doesn’t Work

If you get an error saying winget is not recognized, you may need to update Windows or install the App Installer from the Microsoft Store. Alternatively, use the manual installation method below.

🖥️ Windows Installation (Alternative: Manual Download)

If Winget doesn’t work on your system:

  1. Visit the official uv releases page:
    https://github.com/astral-sh/uv/releases

  2. Download the Windows installer:
    Look for the latest release and download the file named like uv-x86_64-pc-windows-msvc.zip

  3. Extract the ZIP file to a location like C:\Program Files\uv\

  4. Add uv to your system PATH:

    • Press Windows Key and search for “Environment Variables”
    • Click “Edit the system environment variables”
    • Click “Environment Variables” button
    • Under “System variables”, find and select “Path”, then click “Edit”
    • Click “New” and add: C:\Program Files\uv\
    • Click “OK” on all windows

  5. Open a new PowerShell window and verify: uv --version

Part 2: Visual Studio Code (VS Code)

What is VS Code?

Visual Studio Code (commonly called VS Code) is a free, open-source code editor created by Microsoft. It’s currently the most popular development tool in the world, used by millions of developers.

While uv runs your code, VS Code is where you write your code.

Why VS Code?

  1. Popular and Well-Supported

    • Massive community means tons of tutorials, extensions, and help available
    • Frequent updates and bug fixes from Microsoft and the community
    • If you search “how to do X in Python,” you’ll find VS Code examples

  2. Lightweight and Fast

    • Opens quickly and doesn’t slow down your computer
    • Unlike heavy IDEs, it starts in seconds, not minutes

  3. Extensible

    • You can install “Extensions” to add features
    • Only install what you need, keeping it fast and clean
    • Thousands of free extensions available

  4. AI-Ready

    • Best support for AI coding assistants like GitHub Copilot, Cursor, and Cody
    • Many AI-powered forks now exist (like Cursor) because VS Code’s architecture is so good

  5. Industry Standard

    • Used across all programming languages (Python, JavaScript, Go, Rust, etc.)
    • Learning VS Code means you have a transferable skill

Understanding Your Options: Text Editors vs. IDEs

You’ll often hear these terms thrown around. Let’s clarify what the different types of development tools are:

Simple Text Editor (e.g., Notepad, TextEdit)

  • Just lets you type text
  • No syntax highlighting, no error checking, no help
  • Like writing code with a pencil and paper
  • Not suitable for serious development

Full IDE (Integrated Development Environment, e.g., PyCharm, Visual Studio)

  • A massive, all-in-one application
  • Includes everything: compiler, debugger, database tools, GUI builders, testing frameworks, etc.
  • Heavy and complex—like carrying a complete workshop in a truck
  • Pros: Everything built-in, very powerful, works out of the box
  • Cons: Slow to start, uses lots of memory (1-2GB+), overwhelming for beginners, limited customization

VS Code: The Extensible Editor (Our Choice)

  • Starts as a lightweight text editor
  • You add Extensions to turn it into an IDE
  • Think of it like a smartphone: starts simple, but you install apps (extensions) for what you need
  • Pros: Fast, highly customizable, clean interface, grows with you, modern UI
  • Cons: Requires some initial setup (which we’ll do together)
  • Best for: Beginners to professionals, anyone who wants flexibility

Neovim: The Power User’s Editor

Neovim (often written as nvim) is a modern, highly customizable terminal-based text editor. It’s a fork and improvement of the classic Unix editor Vim.

NoteWhat is Neovim?

Neovim is a text editor at its core, but experienced developers configure it to function like a full IDE. It runs entirely in the terminal (no graphical interface by default) and uses keyboard shortcuts exclusively—no mouse needed.

Neovim Characteristics:

  • Terminal-based: Runs in your command line, not as a graphical application - Keyboard-driven: Everything controlled through keyboard commands (steep learning curve) - Infinitely customizable: You can configure it to do anything, but you must configure it yourself - Lightweight: Uses minimal system resources (10-50MB of RAM) - Pros: Blazing fast, works over SSH, ultimate flexibility, impressive once mastered - Cons: Steep learning curve (weeks to months), requires significant configuration, not beginner-friendly

Who uses Neovim? - Senior developers who work primarily in terminals - System administrators managing remote servers - Programmers who prefer keyboard-only workflows - Developers who enjoy tinkering with their tools

Should beginners use Neovim? Not for this course. While Neovim is powerful, it would take weeks just to learn the basics. We need to focus on learning Python and AI, not memorizing editor commands. However, after you’re comfortable with programming, Neovim is an excellent tool to explore.

TipThe Sweet Spot for Learning

VS Code gives you 90% of an IDE’s power with 10% of the complexity. For AI development and learning Python, this is perfect. You can always switch to Neovim later if you want—many professionals use both tools for different situations.

Part 4: Getting Started with uv - Your First Project

Now that we have everything installed, let’s learn how to actually use uv to create and manage Python projects.

Understanding the uv Workflow

The typical workflow with uv looks like this:

  1. Create a project → uv sets up the folder structure and pyproject.toml
  2. Add dependencies → uv downloads packages and updates your configuration
  3. Write code → You develop in VS Code
  4. Run your code → uv ensures the right environment is active

Let’s walk through each step.

Step 1: Installing Python with uv

Before we create projects, let’s install Python itself using uv.

Check available Python versions:

uv python list

This shows all Python versions available for installation.

Install Python 3.12 (recommended for AI/ML):

uv python install 3.12

Verify the installation:

uv python list

You should see Python 3.12 marked as installed.

NoteWhy Python 3.12?

Python 3.12 is the latest stable version with excellent performance improvements (up to 5% faster than 3.11) and is fully compatible with modern AI/ML libraries like PyTorch, TensorFlow, and scikit-learn.

Step 2: Creating Your First Project

Let’s create a simple AI project to practice with.

Create a project directory and initialize it:

mkdir my-first-ai-project
cd my-first-ai-project
uv init -p 3.13

Let’s break down what we just did: - mkdir my-first-ai-project creates a new folder - cd my-first-ai-project navigates into that folder - uv init -p 3.13 initializes a uv project with Python 3.13

The -p 3.13 flag tells uv to use Python 3.13 specifically. If you don’t have Python 3.13 installed, uv will automatically download and install it for you.

This creates the following structure:

my-first-ai-project/
├── pyproject.toml       # Project configuration
├── README.md            # Project documentation
├── .python-version      # Specifies Python version (3.13)
└── hello.py            # Sample Python file

Open it in VS Code:

code .

The . means “open the current directory in VS Code.”

Step 3: Understanding pyproject.toml

Open the pyproject.toml file in VS Code. You’ll see something like:

[project]
name = "my-first-ai-project"
version = "0.1.0"
description = "Add your description here"
readme = "README.md"
requires-python = ">=3.12"
dependencies = []

[build-system]
requires = ["hatchling"]
build-backend = "hatchling.build"

This is your project’s configuration file. Notice: - name: Your project name - version: Your project version (following semantic versioning) - requires-python: Minimum Python version required - dependencies: Empty list (we’ll add packages here)

Step 4: Adding Dependencies (Packages)

Let’s add PyTorch to our project, along with Jupyter for interactive development.

Add PyTorch:

uv add torch torchvision

Add Jupyter tools for interactive notebooks:

uv add ipykernel jupyter notebook jupyterlab

Watch what happens: 1. uv downloads all packages and their dependencies 2. uv automatically adds them to dependencies in pyproject.toml 3. uv creates a uv.lock file that pins exact versions

Check your pyproject.toml again:

[project]
name = "my-first-ai-project"
version = "0.1.0"
description = "Add your description here"
readme = "README.md"
requires-python = ">=3.13"
dependencies = [
    "torch>=2.1.0",
    "torchvision>=0.16.0",
    "ipykernel>=6.29.0",
    "jupyter>=1.0.0",
    "notebook>=7.0.0",
    "jupyterlab>=4.0.0",
]
NoteUnderstanding Jupyter Packages
  • ipykernel: Allows Jupyter to use your virtual environment as a kernel
  • jupyter: The Jupyter metapackage that includes common tools
  • notebook: The classic Jupyter Notebook interface
  • jupyterlab: The modern, feature-rich Jupyter Lab interface (recommended)
TipDevelopment Dependencies

For packages you only need during development (like testing tools), use:

uv add --dev pytest black ruff

These get added to [project.optional-dependencies] instead of dependencies.

Step 5: Running Python Code

There are three main ways to work with Python in your uv project.

Method 1: Using uv run (Recommended for Scripts)

Create a simple test file. In VS Code, create a new file called test_pytorch.py:

import torch

print(f"PyTorch version: {torch.__version__}")
print(f"CUDA available: {torch.cuda.is_available()}")

# Create a simple tensor
x = torch.tensor([1, 2, 3, 4, 5])
print(f"Tensor: {x}")
print(f"Mean: {x.float().mean()}")

Run it with uv:

uv run test_pytorch.py

The uv run command automatically: - Activates the virtual environment - Ensures all dependencies are installed - Runs your Python script

Expected output:

PyTorch version: 2.1.0
CUDA available: False
Tensor: tensor([1, 2, 3, 4, 5])
Mean: 3.0

Method 2: Using Jupyter Lab (Recommended for AI/ML Development)

Jupyter Lab is the modern interface for interactive Python development. It’s perfect for AI/ML work because you can: - Run code in cells and see results immediately - Visualize data and models inline - Document your work with markdown - Experiment without writing full scripts

Start Jupyter Lab:

uv run jupyter lab

This will: 1. Start the Jupyter Lab server 2. Automatically open your browser to http://localhost:8888 3. Show your project files in the left sidebar

Create a new notebook: 1. Click the “+” button or go to File → New → Notebook 2. Select “Python 3 (ipykernel)” as your kernel 3. Start coding!

Test PyTorch in a notebook cell:

import torch

print(f"PyTorch version: {torch.__version__}")

# Create a tensor
x = torch.tensor([[1, 2], [3, 4]])
print(x)

Press Shift+Enter to run the cell.

To stop Jupyter Lab: Press Ctrl+C in the terminal and confirm.

NoteJupyter Notebook vs Jupyter Lab

You can also use the classic Jupyter Notebook interface:

uv run jupyter notebook

However, Jupyter Lab is recommended as it’s more modern, faster, and has better features like a file browser, terminal, and multiple panes.

Step 7: A Complete Example Workflow

Let’s put it all together with a real example. We’ll create a simple image classification project with Jupyter Lab.

1. Create the project:

mkdir image-classifier
cd image-classifier
uv init -p 3.13

2. Add necessary packages:

uv add torch torchvision pillow matplotlib
uv add ipykernel jupyter notebook jupyterlab
uv add --dev pytest ruff black

3. Open in VS Code:

code .

4. Start Jupyter Lab:

uv run jupyter lab

5. Create a new notebook (File → New → Notebook) and add this code:

import torch
import torchvision
from torchvision import transforms
from PIL import Image
import matplotlib.pyplot as plt

print(f"PyTorch version: {torch.__version__}")
print(f"Torchvision version: {torchvision.__version__}")

# Load a pre-trained model
model = torchvision.models.resnet18(pretrained=True)
model.eval()

print("✅ Model loaded successfully!")
print(f"Model architecture: ResNet18")

6. Run the cell by pressing Shift+Enter

7. Create a classification function in a new cell:

def classify_image(image_path):
    """Classify an image using ResNet18"""
    
    # Define image transformations
    transform = transforms.Compose([
        transforms.Resize(256),
        transforms.CenterCrop(224),
        transforms.ToTensor(),
        transforms.Normalize(
            mean=[0.485, 0.456, 0.406],
            std=[0.229, 0.224, 0.225]
        )
    ])
    
    # Load and transform image
    img = Image.open(image_path)
    img_tensor = transform(img).unsqueeze(0)
    
    # Make prediction
    with torch.no_grad():
        output = model(img_tensor)
    
    # Get top prediction
    _, predicted = torch.max(output, 1)
    
    # Display image
    plt.figure(figsize=(8, 6))
    plt.imshow(img)
    plt.title(f"Predicted class: {predicted.item()}")
    plt.axis('off')
    plt.show()
    
    return predicted.item()

print("✅ Classification function ready!")

Step 8: Working with Existing Projects

When you clone a project from GitHub or receive one from a colleague:

1. Navigate to the project folder:

cd existing-project

2. Install all dependencies:

uv sync

This reads pyproject.toml and uv.lock and installs exactly the same versions the original developer used. This ensures “it works on my machine” becomes “it works on everyone’s machine.”

3. Start developing:

code .

Best Practices

Tipuv Best Practices
  1. Always use uv run for scripts to ensure correct environment
  2. Commit pyproject.toml and uv.lock to Git - never commit .venv/
  3. Use version ranges in dependencies (torch>=2.1.0) for flexibility
  4. Separate dev dependencies using --dev flag
  5. Run uv sync after pulling changes from Git
  6. Use uv lock --upgrade periodically to get security updates
  7. One project = one virtual environment - don’t share environments between projects

Troubleshooting Common Issues

Issue 1: “uv: command not found”

Solution: Restart your terminal or add uv to PATH (see installation section)

Issue 2: “Python version not found”

Solution: Install the required Python version:

uv python install 3.12

Issue 3: Package installation fails

Solution: Check your internet connection and try:

uv cache clean
uv add torch --reinstall

Issue 4: VS Code doesn’t recognize Python

Solution: 1. Press Ctrl+Shift+P (Windows/Linux) or Cmd+Shift+P (Mac) 2. Type “Python: Select Interpreter” 3. Choose the interpreter from .venv folder

Issue 5: Import errors when running code

Solution: Make sure you’re using uv run or have activated the virtual environment:

uv run python your_script.py
