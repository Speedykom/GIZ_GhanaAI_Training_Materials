C. Looping Two Lists (zip)

AI data often comes in separate lists (e.g., images and labels).
Use zip() to loop them together like a zipper on a jacket.

names = ["Kwame", "Ama", "Yaw"]
scores = [85, 92, 78]

# Loop simultaneously
for name, score in zip(names, scores):
    print(f"{name} scored {score}")
Kwame scored 85
Ama scored 92
Yaw scored 78

You are not limited to just two lists! You can zip three, four, or as many iterables as you need.

names = ["Alice", "Bob", "Charlie"]
scores = [85, 92, 78]
roles = ["Admin", "User", "Moderator"]

# Looping through three lists simultaneously
for name, score, role in zip(names, scores, roles):
    print(f"{name} ({role}) scored {score}")
Alice (Admin) scored 85
Bob (User) scored 92
Charlie (Moderator) scored 78

In Machine Learning, we almost always have two separate lists: Features (the inputs like image pixels) and Labels (the answers like “cat” or “dog”). We use zip() to pair them together so the computer can learn that this specific input belongs to that specific answer.

Why this matters: In data science, you’ll often have parallel arrays (features and labels, timestamps and values). zip() is essential.

Critical Edge Cases with zip()

WarningWhat happens with unequal lengths?

zip() stops at the shortest list! Data is silently lost.

names = ["Kwame", "Ama", "Yaw", "Akosua"]  # 4 items
scores = [85, 92]                           # 2 items

for name, score in zip(names, scores):
    print(f"{name}: {score}")

# Output: Only Kwame and Ama!
# Yaw and Akosua are IGNORED
Kwame: 85
Ama: 92

Safe Alternative: Use zip_longest() from itertools if you need all items:

from itertools import zip_longest

names = ["Kwame", "Ama", "Yaw", "Akosua"]
scores = [85, 92]

for name, score in zip_longest(names, scores, fillvalue=0):
    print(f"{name}: {score}")

# Output includes all names, missing scores filled with 0
Kwame: 85
Ama: 92
Yaw: 0
Akosua: 0

What can you zip()?

Answer: Any iterable! Not just lists.

set1 = {"Python", "JavaScript", "Go"}

    print(f"{lang}: {role}")

C3
TipBest Practice
  • Use zip() when you’re certain both iterables are the same length
  • Use zip_longest() when lengths might differ and you need all data
  • Never use zip() with sets in production code (unpredictable order)
  • Always validate data lengths before zipping in real applications

3. List Comprehensions (The “Killer Feature”)

This is the most important tool for data cleaning. It replaces .map() and .filter().

Syntax: [expression for item in list if condition]

The Bridge: Traditional Loop vs. Comprehension

Before looking at the syntax, let’s look at why we use comprehensions. Suppose we want to multiply every number in a list by 5.

Here is the traditional way using a standard for loop and .append():

nu_list = [2, 3, 7, 11, 3, 0.312, 45, 78, 32, 12, 71]

# 1. Create an empty list
prod = []

# 2. Loop through the original list
for item in nu_list:
    # 3. Do the math
    ans = item * 5
    # 4. Append to the new list
    prod.append(ans)

print("Traditional Loop:", prod)
Traditional Loop: [10, 15, 35, 55, 15, 1.56, 225, 390, 160, 60, 355]

Now, here is the exact same logic written as a List Comprehension. It collapses those 4 lines of code into a single, elegant line:

# [expression FOR item IN iterable]
prod2 = [item * 5 for item in nu_list]

print("List Comprehension:", prod2)
List Comprehension: [10, 15, 35, 55, 15, 1.56, 225, 390, 160, 60, 355]

Example 1: The .map() Equivalent

Multiply every number by 2.

nums = [1, 2, 3, 4]

# JS: const doubled = nums.map(x => x * 2)
doubled = [x * 2 for x in nums]

print(doubled)
[2, 4, 6, 8]

Example 2: The .filter() Equivalent

Keep only even numbers.

# JS: const evens = nums.filter(x => x % 2 === 0)
evens = [x for x in nums if x % 2 == 0]

print(evens)
[2, 4]

Example 3: Chaining (Filter + Map)

In JavaScript: nums.filter(x => x % 2 === 0).map(x => x * 2)

# Filter AND transform in one line
result = [x * 2 for x in nums if x % 2 == 0]

print(result)
[4, 8]
NoteReal-World AI Connection: Feature Scaling

Before feeding data into an AI model, we often need to “scale” it (e.g., converting all numbers to be between 0 and 1). List comprehensions are the fastest way to apply a mathematical formula to thousands of data points instantly.

WarningReadability First

If your list comprehension becomes too complex (nested conditions, multiple transformations), break it into a regular loop or multiple steps. Readability > cleverness.

From For Loop to List Comprehension: Walking Backwards

The key skill is reverse engineering a for loop into a comprehension. Let’s learn the pattern.

The Translation Pattern

Every list comprehension follows this structure:

result = [WHAT_TO_KEEP for ITEM in SOURCE if CONDITION]

Step-by-step translation:

  1. Find what’s being appended → This becomes WHAT_TO_KEEP
  2. Find the loop variable → This becomes ITEM in SOURCE
  3. Find any if statement → This becomes the if CONDITION

Worked Example: Temperature Converter

Let’s convert this loop together:

# Original For Loop
celsius_temps = [0, 10, 20, 30, 40]
fahrenheit = []

for temp in celsius_temps:
    if temp >= 20:  # Only convert warm temperatures
        converted = temp * 9/5 + 32
        fahrenheit.append(converted)

print(fahrenheit)
[68.0, 86.0, 104.0]

Translation Steps:

# Step 1: What's being appended?
#         → converted = temp * 9/5 + 32

# Step 2: What's the loop?
#         → for temp in celsius_temps

# Step 3: What's the condition?
#         → if temp >= 20

# Combine them:
fahrenheit = [temp * 9/5 + 32 for temp in celsius_temps if temp >= 20]

Let’s verify they produce the same result:

# The comprehension version
celsius_temps = [0, 10, 20, 30, 40]
fahrenheit = [temp * 9/5 + 32 for temp in celsius_temps if temp >= 20]

print(fahrenheit)  # [68.0, 86.0, 104.0]
[68.0, 86.0, 104.0]
TipReading Comprehensions Aloud

Try reading it in English: “Give me temp * 9/5 + 32 for each temp in celsius_temps if temp >= 20”

4. Lambda Functions: A Brief Intro

In JavaScript, you often use arrow functions for quick operations: (x) => x * 2. Python’s equivalent is the lambda function.

We use lambdas for quick, one-line operations where defining a full function with def would be overkill. You’ll see these most often when sorting data or using specific libraries.

# The Syntax: lambda arguments: expression
double = lambda x: x * 2
print(double(5)) # 10

Common Use: Sorting Data

You have a list of students and want to sort by age. The key parameter expects a function that tells it how to sort.

students = [
    {"name": "Kwame", "age": 22},
    {"name": "Ama", "age": 19},
    {"name": "Yaw", "age": 25}
]

# Use lambda to tell sort() to look at the "age" key
students.sort(key=lambda s: s["age"])
print(students)
[{'name': 'Ama', 'age': 19}, {'name': 'Kwame', 'age': 22}, {'name': 'Yaw', 'age': 25}]
NoteMoving Forward

We’ll dive much deeper into advanced Lambda patterns (like filtering, data mapping, and multi-level sorting) in Lesson 3.

Real-World Example: Processing API Data

Here’s how lambda functions are used in actual data engineering:

# You fetch weather data from an API
weather_data = [
    {"city": "Accra", "temp": 32, "humidity": 75},
    {"city": "Kumasi", "temp": 28, "humidity": 80},
    {"city": "Tamale", "temp": 35, "humidity": 60},
    {"city": "Cape Coast", "temp": 30, "humidity": 85}
]

# Find the hottest city
hottest = max(weather_data, key=lambda city: city["temp"])
print(f"Hottest city: {hottest['city']} at {hottest['temp']}°C")

# Find cities with high humidity (>70%)
humid_cities = list(filter(lambda c: c["humidity"] > 70, weather_data))
print(f"Humid cities: {[c['city'] for c in humid_cities]}")

# Extract just the temperatures
temps = list(map(lambda c: c["temp"], weather_data))
print(f"All temperatures: {temps}")
Hottest city: Tamale at 35°C
Humid cities: ['Accra', 'Kumasi', 'Cape Coast']
All temperatures: [32, 28, 35, 30]
WarningThe “Pythonic” Way

While the above code works, experienced Python developers would write:

# More readable and Pythonic
hottest = max(weather_data, key=lambda c: c["temp"])  # Lambda is good here
humid_cities = [c for c in weather_data if c["humidity"] > 70]  # Comprehension
temps = [c["temp"] for c in weather_data]  # Comprehension

# Rule of thumb: Use lambda for key functions, comprehensions for transformations

We use def to define functions. But for AI work, you need to understand Dynamic Arguments.

A. Default Arguments

You can set default values directly in the function definition.

def greet(name, greeting="Akwaaba"):
    return f"{greeting}, {name}!"

print(greet("Kofi"))            # Uses default
print(greet("Kofi", "Hello"))   # Overrides default
Akwaaba, Kofi!
Hello, Kofi!

B. *args and **kwargs (The “Rest” Parameters)

AI libraries (like Transformers) use these to accept endless configuration settings.

  • *args: Collects extra positional arguments into a Tuple (like JS ...args)
  • **kwargs: Collects extra keyword arguments into a Dictionary
def build_model(name, **config):
    print(f"Building: {name}")
    print("Config:", config)  # It's just a dict!

# We can pass ANY arguments we want
build_model("ChatBot", layers=12, speed="fast", gpu=True)
Building: ChatBot
Config: {'layers': 12, 'speed': 'fast', 'gpu': True}
TipReal-World AI Example: The “Config Builder”

In AI libraries like HuggingFace Transformers, models take dozens of settings (dropout rates, attention heads, etc.). Instead of writing a function with 50 parameters, developers use **kwargs.

# A realistic look at how AI models are initialized
def initialize_ai_model(model_name, **hyperparameters):
    print(f"Loading weights for: {model_name}")
    for param, value in hyperparameters.items():
        print(f"  -> Setting {param} to {value}")

# The user can pass only the settings they want to change
initialize_ai_model("BERT-Base", learning_rate=2e-5, batch_size=32, layers=12)

Why this matters: When you use HuggingFace or PyTorch, you’ll constantly pass configuration objects this way.

def super_function(*args, **kwargs):
    print("Positional arguments (tuple):", args)
    print("Keyword arguments (dict):", kwargs)

super_function(1, 2, 3, mode="turbo", verbose=True)
Positional arguments (tuple): (1, 2, 3)
Keyword arguments (dict): {'mode': 'turbo', 'verbose': True}

6. Modules: Organizing Code

In JavaScript, you use export and import (or module.exports).
In Python, every file is automatically a module.

A. Importing

Assume we have a file named math_utils.py:

# Import the whole file
import math_utils
math_utils.calculate()

# Import specific function (Like destructuring in JS)
from math_utils import calculate
calculate()

# Alias (Renaming) - Very common in data science
import pandas as pd
import numpy as np

B. The if __name__ == "__main__": Block

You will see this everywhere. It’s a guard block.

Code inside this block only runs if you execute the file directly (e.g., python file.py). It does not run if you import the file into another script.

def main():
    print("Running main logic...")

if __name__ == "__main__":
    main()

Use case: You can write reusable functions at the top of a file, and testing code at the bottom that only runs when you test the file directly.
